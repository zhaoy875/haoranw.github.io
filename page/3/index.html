关于String的一些细节

Ⅰ.当字符串内含有转义字符时，使用.length返回的是转义后字符串的长度。

var str="\x41";
alert(str.length);//1

同时，即使字符串中包含双字节字符（比如汉字），每个也只算一个字符。

var str="王浩然";
alert(str.length);//3

Ⅱ.字符串一旦创建，它的值不可改变。我们在改变变量保存字符串的值时，实际上一个创建新字符串、赋值、删除原字符串的过程。所以在某些低版本浏览器中，字符串拼接效率较低。

var str="Hello";
str+="World"

过程是得到Hello+World的长度，即10。然后创建一个长度为10的字符串，之后输入"Hello"和"World"。最后销毁原字符串"Hello"和字符串"World"。

Ⅲ.null和undefined没有toString()方法，因为我们在对一个不确定的值使用toString()方法 时，存在报错的风险。
但是另一种转换方法String()则没有这个顾虑，因为String()的转换规则是：
1.如果可以使用toString()方法，则调用并返回。
2.如果值是null或undefined，则对应返回"null"或"undefined"。

Ⅳ.当传入参数为负值时：
slice()将负值与字符串长度相加；
sustr()第一个参数为负，则加上字符串的长度。第二个参数为负，转换为0；
substring()会将所有的副参数转为0。它还有一个行为就是，在面对两个参数时，永远是从小的参数开始，到大的参数对应位置结束。而不是从第一个参数开始。

Ⅴ.字符串匹配：
1.match()返回的是一个数组。

var str="dog,rat,cat,hat,mat";
var _match=str.match(/.at/);
console.log(_match);


//console:
["rat", index: 5, input: "dog,rat,cat,hat,mat"]
0:"rat"
index:5
input:"dog,rat,cat,hat,mat"
length:1

依次是匹配到的字符串片段、其开始的下标、输入的字符串。

search()返回的是第一个匹配项的索引。

replace()约定了一个特殊标记符$：
$i (i:1-99) ：表示从左到右正则子表达式所匹配的文本。
$&：表示与正则表达式匹配的全文本。
$`(切换技能键)：表示匹配字符串的左边文本。
$’(单引号)：表示匹配字符串的右边文本。
$$：表示$。

"boy & girl".replace(/(\w+)\s*&\s*(\w+)/g,"$2 & $1") //girl & boy
"boy".replace(/\w+/g,"$&-$&") // boy-boy


var str="javascript"
str.replace(/script/,"$& != $`") //javascript != java
str.replace(/java/,"$&$' is ") // javascript is script

从创建一个字符串说起

首先我们可能都知道创建一个字符串通常有三种写法：

var str1="wanghaoran";
var str2=String("wanghaoran");
var str3=new String("wanghaoran");

我们对每一个str执行typeof，可以得到如下结果：

typeof str1;
// 'string'
typeof str2;
// 'string'
typeof str3;
// 'object'

前两种方法创建的字符串都属于Primitive String ，即原始数据类型中的原始类型字符串，返回的自然是string。

而第三种方法使用了构造函数创建了一个String Object，所以它的类型是对象，即Object

那么问题来了，为什么一个原始类型的字符串能够调用各种字符串方法或者查看属性呢？这不是对象能做的事吗？

要了解这个问题的答案就要从源头着手。首先根据ES发布的规范：

JS在运行到形如PrimitiveType.method即处理某个基本类型的某方法时，会调用CheckObjectCoercible(baseValue)方法，在此方法中会进行ToObject运算。

针对String类型的ToObject运算的描述如下：

Create a new String object whose [[PrimitiveValue]] internal property is set to the value of the argument.

过程中创建了一个对象，并且将这个字符串的内容复制给这个对象。

但是这个包装对象是临时的，在操作完成时就会释放，下次再重新访问时会重新建立。所以虽然我们可以像对象那样，在原始数据类型的值上调用各种内置的对象方法，但是对原始类型的数值进行自定义方法或者属性时，并不能得到保存。

var person="wanghaoran";
person.sex="male";
person.callName=function(){
    alert(this);
}
console.log(person.sex);//undefined
person.callName();//报错...is Not a function

当然以下方法是没有问题的：

var person=new String("wanghaoran");
person.sex="male";
person.callName=function(){
    alert(this);
}
console.log(person.sex);//male
person.callName();//弹出wanghaoran
